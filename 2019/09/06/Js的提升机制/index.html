JavaScript的提升机制

我在看到一本书的如下代码时有了困惑：

此时上面foo()的调用结果为1。

按照我之前对变量提升的理解来看，function foo(){}这个函数声明以及var foo的变量声明都会提升，那么谁提升的多呢，因此我将foo()的调用改为了如下代码： 

console.log(foo);
function foo(){
    console.log(1);
}
//输出结果为foo所在的函数声明

因此联想到在编写JavaScript代码时，后赋值的语句会覆盖先赋值的语句的原则，个人认为var的变量声明在foo的函数声明之前，也就是说变量声明的提升大于函数声明的提升。

接下来我进行了各种代码的测试编写，怎么也搞不明白为什么说函数声明的提升大于变量声明的提升。

好吧，最后我懵了。

一直到我了解到了JavaScript的提升机制叫做hoisting机制，而这个hoisting机制就十分神奇了。

谜底揭晓了

hoisting机制含义解释简单实例
1、我是变量声明，我会被提升在作用域顶端!
　　　　eg: var a;
2、我是变量定义，我的声明部分会被提升，赋值部分不会被提升!
　　　　eg: var a = ‘hello’;
3、我是函数定义，或者叫我函数表达式。其实我就是变量定义，只不过恰好被赋值的类型是函数，所以也只提升变量名，不提升函数值！
　　　　eg: var a = function (){
　　　　　　　　console.log(‘hello’);
　　　　　　};
4、我是函数声明，所以我全部被提升了，包括函数名和函数体。另外，我的优先级比变量声明要高，名字和我相同的变量声明会被忽略！
　　　　eg: function a (){
　　　　　　　　console.log(‘hello’);
　　　　　　};

对于同名的函数声明和变量声明，Javascript采用的是声明阶段忽略变量声明，赋值阶段覆盖原则

好吧，一切的问题拨开迷雾见云天了。

但是调用的位置不同，就需要考虑到这个赋值覆盖原则啦。

var foo;
function foo(){
    console.log(1);
}
foo=function(){
    console.log(2);
}
foo();//这时的结果就是2啦。

如果你想更加理解这个机制，还是自己多敲代码进行学习吧。
对了，推荐一本书《你不知道的JavaScript》，我的疑问就是从这本书里看到的，这一系列的书很强。
